Use Cloudflare's "1.1.1.1 for Families" DNS-based blocking?



Read Cloudflare's post about "1.1.1.1 for Families" for more info.
When enabled, this plugin will first check to see if the domain the image is coming from is a known adult content provider before doing scanning via AI.
There are advantages of enabling DNS blocking as an additional check:
  • All videos are checked because they are simply blocked by domain.
  • Similarly, provides a second check on images.
  • Done as a pre-check, so if it is blocked the request to get the image is cancelled and the slower AI doesn't need to run.
There are, however, some disadvantages:
  • Requires a call to Cloudflare's servers with the hostname (e.g. "google.com") of items being checked from your browsing session.
  • Because blocking is performed based on hostname, it can sometimes block all media for a website with many types of content.
Please make the choice that is right for you!
